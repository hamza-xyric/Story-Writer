## Overview

Expert Agents are comprehensive AI specialists, each embodying the knowledge, practices, and mindset of a top 1% professional in their field. Unlike task-specific skills, Expert Agents provide:

- **Deep domain expertise** across their entire specialty
- **Best practices** from industry leaders (Google, FAANG, OWASP)
- **Decision-making frameworks** for complex technical choices
- **Quality standards** representing elite professional expectations
- **Mentorship-level guidance** for skill development

### EXPERT-26: Brainstormer (v1.0)
**File**: `EXPERT-26-brainstormer.md`
**Expertise**: Pre-vision idea exploration, Hard Questions framework, contradiction detection
**Best For**: Transforming raw ideas into structured brainstorm documents before vision creation
**Architecture**: 5-phase workflow with Hard Questions (WHY/WHO/WHAT/HOW/WHAT-IF)
**Input**: Raw product ideas, notes, sketches (any format)
**Output**: brainstorm.md with validated assumptions and VISION_READY marker
**Downstream**: EXPERT-20 (enters reduced-questions mode when brainstorm.md exists)

### EXPERT-20: Vision Generator (v2.1)
**File**: `EXPERT-20-vision-generator.md`
**Expertise**: Product ideation, vision document creation (product-focused, NOT business model)
**Best For**: Transforming rough product ideas into comprehensive vision documents
**Architecture**: 4-phase workflow with two entry modes
**Upstream**: EXPERT-26 (Brainstormer)
**Invokes**: BI-02 Question Framework
**Input**: Product idea, notes, OR brainstorm.md from EXPERT-26
**Output**: 10-section Vision Document (reduced from 13, no business content)
**Entry Modes**:
- **With Brainstorm**: 4 reduced questions (brainstorm.md provides context)
- **Without Brainstorm**: 7 full questions (legacy mode)
**Focus**: 60% Product Concept, 30% Problem/Solution, 10% Strategic Intent

### EXPERT-21: PRD Generator (v2.0)
**File**: `EXPERT-21-prd-generator.md`
**Expertise**: Product requirements, feature definition, inspiration-focused competitive analysis
**Best For**: Transforming vision documents into actionable PRDs
**Architecture**: 4-phase workflow with 7 question categories
**Invokes**: BI-02 Question Framework
**Input**: Vision Document
**Output**: Master PRD with Quality Metrics (not revenue metrics)
**Key Change**: Competitive analysis reframed as "What We Can Learn From Others"
**Downstream**: EXPERT-27 (Technical Architecture Generator)

### EXPERT-27: Technical Architecture Generator (v1.0)
**File**: `EXPERT-27-technical-architecture-generator.md`
**Expertise**: System architecture, technology decisions, ADR creation, component design
**Best For**: Transforming PRDs into comprehensive Technical Architecture documents
**Architecture**: 4-phase workflow with 6 questions
**Type**: Generator (comprehensive output)
**Invokes**: EXPERT-03 (Software Architect) for ADR creation
**Input**: Master PRD from EXPERT-21
**Output**: Technical-Architecture.md with ADRs, Epic-Component Mapping Matrix
**Key Features**:
- Modular Monolith First philosophy
- ADR-driven decision making
- Epic-component mapping for downstream skills
- 12 required sections with blocking validation
**Upstream**: EXPERT-21 (PRD Generator)
**Downstream**: EXPERT-22 (Epic Generator)

### EXPERT-22: Epic Generator (v2.0)
**File**: `EXPERT-22-epic-generator.md`
**Expertise**: Feature grouping, MoSCoW distribution, hypothesis-driven scoping, complexity scoring
**Best For**: Breaking PRDs into comprehensive epic outlines
**Architecture**: 4-phase workflow with 5 questions
**Type**: Generator (bulk output)
**Invokes**: BI-02 Hypothesis Framework
**Input**: Master PRD + Technical Architecture from EXPERT-27
**Output**: Epic Overview with MoSCoW (60/20/20), Hypothesis, Complexity Scores
**Upstream**: EXPERT-27 (Technical Architecture Generator)

### EXPERT-23: Epic PRD Generator (v2.0)
**File**: `EXPERT-23-epic-prd-generator.md`
**Expertise**: Feature specification, BDD acceptance criteria, ADR flags
**Best For**: Creating detailed Epic PRDs with full feature specs
**Architecture**: 4-phase workflow with 4 questions
**Type**: Individual (one epic at a time)
**Invokes**: EXPERT-03 (for ADRs), BI-02
**Input**: Epic outline from EXPERT-22
**Output**: Detailed Epic PRD with Given/When/Then acceptance criteria

### EXPERT-14: Story Spec Generator (v2.0)
**File**: `EXPERT-14-story-spec-generator.md`
**Expertise**: Technical design, architecture patterns, ADR flagging, API contracts
**Best For**: Creating technical specifications from user stories
**Architecture**: 4-phase workflow with 5 questions
**Type**: Individual (one story at a time)
**Invokes**: EXPERT-03 (for ADRs), BI-02
**Input**: User Story from EXPERT-13
**Output**: Technical Specification with architecture patterns, design system cross-refs

### EXPERT-15: Task Generator (v2.0)
**File**: `EXPERT-15-task-generator.md`
**Expertise**: TDD task ordering, engineer assignment, MoSCoW enforcement
**Best For**: Breaking tech specs into implementation tasks
**Architecture**: 4-phase workflow with 4 questions
**Type**: Generator (bulk output)
**Invokes**: EXPERT-04 (for TDD), DEV-02 (test patterns)
**Input**: Technical Specification from EXPERT-14
**Output**: Task Breakdown with TDD ordering (ARCH → TEST → IMPL → INTG)

### EXPERT-16: Task Spec Generator (v2.0)
**File**: `EXPERT-16-task-spec-generator.md`
**Expertise**: High-level implementation guidance (NO code examples)
**Best For**: Creating implementation guides for engineers
**Architecture**: 4-phase workflow with 3 questions
**Type**: Individual (one task at a time)
**Invokes**: EXPERT-03, EXPERT-04, DEV-02
**Input**: Task from EXPERT-15
**Output**: HIGH-LEVEL GUIDE ONLY (engineers write actual code)

### Pipeline Standards

All pipeline skills follow:
- **4-Phase Workflow**: Research → Questions → Generation → Validation
- **Blocking Validation**: Must pass all quality checks before output
- **100% Coverage**: No gaps between inputs and outputs
- **Product-Agnostic**: Works for any product type
- **Pipeline Hooks**: Output formatted for downstream consumption

See `SKILL-PIPELINE-PROGRESS.md` for complete pipeline documentation.
